# Brainstorm General

## Overview
Run a consistent workflow that converts ambiguous ideas into validated plans.
Keep sessions practical: clarify intent, generate options, score trade-offs, test assumptions, and commit to an implementation path.

## Workflow

### 1. Clarify the Goal
Collect or confirm:
- Objective (one sentence)
- Target audience or beneficiary
- Timeline and urgency
- Non-negotiable constraints (time, budget, energy, tools)
- Success metric (what outcome proves this worked)

If key context is missing, ask concise targeted questions first.
If the user prefers speed, proceed with explicit assumptions.

### 2. Define Decision Criteria
Before ideation, define how options will be judged.
Use this baseline unless the user provides a custom rubric:
- Impact
- Effort
- Risk
- Confidence
- Reversibility

Use scoring rules in `references/scoring.md`.

### 3. Diverge (Generate Strong Options)
Create 8-20 options (volume based on complexity).
Force variety by using multiple ideation lenses from `references/lenses.md`.
Include at least:
- 2 conservative options
- 2 high-upside options
- 1 low-effort immediate action
- 1 experiment-first option

Avoid obvious restatements. Push for distinct approaches.

### 4. Converge (Select a Direction)
Score top candidates with the rubric.
For each shortlisted option, state:
- Why it can work
- Failure modes
- Earliest disconfirming signal

Select one primary path and one fallback.
If top options are close, prefer the path with faster learning and lower irreversible downside.

### 5. Confidence and Test Plan
Design a staged confidence system:
- Stage 0: desk check (logic and assumptions)
- Stage 1: small test / prototype
- Stage 2: limited rollout
- Stage 3: scaled rollout

Define gates for each stage:
- Entry criteria
- Metric target
- Stop/continue rule
- Owner and timeline

### 6. Implementation Plan
Produce a specific plan with:
- 24-hour next actions
- 7-day milestone plan
- 30-day scale plan (if relevant)
- Risks and mitigations
- Tracking cadence and review checkpoint

Use output shapes from `references/output-templates.md`.

## Domain Adapters
Apply the same workflow to any domain, then tune metrics:
- Tweet/content: hook quality, saves/shares, retention, comments quality
- Website/product direction: activation, conversion, task success, bounce
- Learning/revision: completion rate, recall quality, timed performance
- Interview practice: signal coverage, weak-area closure, mock score trend
- Life/career decisions: regret minimization, reversibility, downside protection

## Quality Bar
Always enforce:
- Clear assumptions and constraints
- Trade-off transparency
- Test-before-scale bias
- Actionable next steps with dates
- No generic motivational filler
