# Anti-Patterns Reference

A synthesis of Uncodixfy's "Normal UI" standards and the premium-frontend aesthetic direction. These are **hard stops** — not style preferences.

---

## The "AI Slop" Test

Before shipping any UI, ask: *"Does this look like it was generated by an AI in 2024?"*

Signals you've failed the test:
- The color palette could be any SaaS product
- The layout is a grid of metric cards with eyebrow labels
- Every card has the same border radius as the modal, the button, and the avatar
- Animations trigger on hover and involve `scale` or `translateY(-4px)`
- There's a gradient somewhere that wasn't explicitly requested
- The font is Inter, Roboto, or Space Grotesk

---

## Layout Anti-Patterns

### ❌ Metric-Card Grid as Default

**Pattern:** Dashboard opens with 4–6 KPI cards in a grid, then a table below.

**Why it fails:** Every SaaS product looks like this. It's not wrong — it's just not thinking. Cards are overused because they're safe.

**Instead:** Start with the user's primary action. What do they need to *do* first? Build toward that, then surface metrics contextually.

---

### ❌ Sticky Sidebar as Default Navigation

**Pattern:** Every page has a left-side nav rail, fixed on scroll, containing 8–12 items.

**Why it fails:** It consumes 240px of content space on every page, including ones that don't need navigation context.

**Normal implementation:** 240–260px sidebar is fine — but earn it. Simple apps should use top nav. Complex nested flows use contextual sidebars.

---

### ❌ Hero Sections in Dashboards

**Pattern:** Dashboard has a full-width banner/hero at the top with an illustration and a headline like "Welcome back, Chintan."

**Why it fails:** Heroes are for landing pages. Dashboard users arrived — they want to work, not be welcomed.

**Instead:** Jump to the user's primary action. Use subtle personalization in context (inside the task flow, not above it).

---

### ❌ Eyebrow Labels + Decorative Copy

**Pattern:** `ANALYTICS OVERVIEW` (all caps, small tracking) above `Track your performance metrics`.

**Why it fails:** The eyebrow label has no information content. The subtitle restates the obvious.

**Instead:** One clear heading that says exactly what the section does. No eyebrows. No decorative subtitles.

## Visual Anti-Patterns

### ❌ Oversized Border Radius Uniformly Applied

**Pattern:** `border-radius: 20–32px` on cards, modals, inputs, buttons, and avatars.

**Why it fails:** Pill-shaped everything reads as toy-like and unrefined. Large radii communicate playfulness — reserve them for playful contexts.

**Normal values:**
- Buttons: 6–8px (or match card)
- Cards: 8–12px
- Modals: 12–16px
- Tags/badges: 4–6px (or pill only if intentional)
- Inputs: 6–8px

---

### ❌ Glassmorphism Floating Shells

**Pattern:** `backdrop-filter: blur(20px)` + semi-transparent background on modals, cards, and sidebars.

**Why it fails:** Readable only against specific backgrounds. Destroys legibility when the background changes. Used to signal "modern" but now signals "2022 design template."

**Instead:** Solid surfaces. Depth through shadows (subtle — max 0.12 alpha). Elevation through z-index, not transparency.

---

### ❌ Soft Corporate Gradients

**Pattern:** `background: linear-gradient(135deg, #667eea, #764ba2)` — or any purple-to-blue gradient on white.

**Why it fails:** This gradient has been used in more Dribbble shots than any other. It communicates "I ran out of ideas."

**Color priority:**
1. Use project/brand colors
2. Use a predefined palette (OKLCH, referenced in color guide)
3. Never invent palette colors

---

### ❌ Gradient Text (`background-clip: text`)

**Pattern:** `background: linear-gradient(...); -webkit-background-clip: text; color: transparent` on headings.

**Why it fails:** Color is for semantic meaning. Gradient text has no meaning — it's pure decoration. Also breaks on Safari in edge cases.

**Instead:** Bold weight + strong color for emphasis. If you need visual interest, use typography (size, weight, tracking) not gradient paint.

---

### ❌ Dramatic Box Shadows

**Pattern:** `box-shadow: 0 20px 60px rgba(0, 0, 0, 0.3)` on cards.

**Why it fails:** Looks ungrounded. Heavy shadows look like the element is floating in space, not sitting on a surface.

**Normal shadow scale:**
- Level 1 (subtle): `0 1px 3px rgba(0,0,0,0.06), 0 1px 2px rgba(0,0,0,0.04)`
- Level 2 (card): `0 4px 12px rgba(0,0,0,0.08)`
- Level 3 (modal): `0 8px 24px rgba(0,0,0,0.12)`
- Max alpha: 0.12

## Animation Anti-Patterns

### ❌ `transform: scale(1.05)` on Hover

**Pattern:** Cards, buttons, and images grow slightly on hover.

**Why it fails:** Scale shifts layout context and draws attention away from content. It signals "I hover, click me" more than the content itself.

**Instead:** Subtle `background-color` shift (4–8% lightness difference). Or a border color change. Reserve scale for image galleries and portfolio items where the signal is intentional.

---

### ❌ `translateY(-4px)` Card Lift

**Pattern:** Cards lift on hover via `transform: translateY(-4px)` + increased shadow.

**Why it fails:** Same as scale — it's expected, overused, and adds no information. Every card in every Bootstrap clone does this.

**Instead:** Border color change + subtle shadow increase. Maximum shadow delta: one level up on the shadow scale.

---

### ❌ Animated Progress/Pipeline Bars with Gradient Fills

**Pattern:** Loading bars or pipeline stages filled with `linear-gradient(90deg, #6366f1, #8b5cf6)`.

**Why it fails:** Purple gradient fill is the #1 indicator of "AI-generated UI." It's everywhere.

**Instead:** Solid brand color. Or a progress indicator that uses the existing design system accent color. Gradients add nothing to a progress bar.

---

## Color Anti-Patterns

### ❌ Pure Neutral Grays

**Pattern:** Text at `#888888`, borders at `#e5e5e5`, backgrounds at `#f5f5f5`.

**Why it fails:** Pure gray reads as "default browser." It has no personality and creates harsh contrast in dark mode.

**Instead:** Tinted neutrals. Add 0.01–0.02 chroma in OKLCH toward the brand hue.
```css
/* ❌ */
--color-muted: hsl(0, 0%, 53%);

/* ✅ */
--color-muted: oklch(53% 0.015 250); /* cool blue tint */
```

---

### ❌ Pure Black Text on White

**Pattern:** `color: #000000; background: #ffffff`.

**Why it fails:** 21:1 contrast ratio is too high — clinically harsh. Reading fatigue increases.

**Instead:** Near-black with slight warm or cool tint:
```css
--color-text: oklch(15% 0.01 250); /* near-black, cool */
--color-background: oklch(99% 0.005 80); /* near-white, warm */
```

## The "Linear/Stripe/GitHub" Test

Before shipping, ask: *"Would this UI fit in Linear, Stripe, or GitHub?"*

These products:
- Use content-driven layouts (not formula-driven)
- Have typography that does most of the visual work
- Use color for meaning, not decoration
- Have subtle, purposeful motion
- Feel functional even before you read the content

If your UI wouldn't survive this comparison, find the weakest element and fix it first.
