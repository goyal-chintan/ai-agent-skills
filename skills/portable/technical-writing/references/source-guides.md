# External Source Guides

Curated references for technical writing best practices. Use these as authoritative sources when making style decisions.

---

## Google Technical Writing

**URL:** https://developers.google.com/tech-writing

Two free courses covering fundamentals:
- **Technical Writing One** — grammar, active voice, clear sentences, lists, paragraphs
- **Technical Writing Two** — organizing documents, drafting, illustrating, self-editing

**When to cite:** Sentence-level clarity, active vs. passive voice, list formatting.

---

## Microsoft Writing Style Guide

**URL:** https://learn.microsoft.com/en-us/style-guide/welcome/

Comprehensive guide for technology content:
- Bias-free communication
- Global-ready writing (localization-friendly)
- Accessibility in documentation
- UI text and error message guidelines

**When to cite:** Inclusive language, UI copy, capitalization and punctuation conventions.

---

## Write the Docs

**URL:** https://www.writethedocs.org/guide/

Community-maintained guide covering:
- Documentation principles and philosophy
- Docs-as-code workflows
- Information architecture for documentation sites
- Starting and maintaining documentation programs

**When to cite:** Documentation strategy, docs-as-code tooling, community best practices.

---

## Plain Language Guidelines

**URL:** https://www.plainlanguage.gov/guidelines/

Rooted in the U.S. Plain Writing Act; applicable to any audience:
- Write for your audience
- Organize information logically
- Use simple words and short sentences
- Use active voice and present tense
- Test your document with real readers

**When to cite:** Simplifying complex prose, government or public-facing documentation, readability standards.

---

## How to Use These References

1. **Don't memorize** — refer to the source when making a specific style decision.
2. **Resolve conflicts** — when guides disagree, prefer the one closest to your audience's context.
3. **Link, don't copy** — reference the URL rather than reproducing their content.
4. **Stay current** — these guides update; check the source if a recommendation seems outdated.
